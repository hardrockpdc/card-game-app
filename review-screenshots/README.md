# Review screenshots (scratch)

Drop screenshots here for Claude to look at when reporting a visual/layout bug.

**Workflow**
1. You add image(s) to this folder (PNG/JPG). Naming is up to you — something
   like `wildround-reveal-overlap.png` helps.
2. Tell Claude what's wrong; Claude reads the images from here.
3. Once the issue is fixed and confirmed, Claude deletes the image(s) — this is a
   temporary holding area, not an asset store.

**Note:** the images themselves are git-ignored (kept local, never committed or
pushed) so they don't bloat the repo. Only this README is tracked, so the folder
always exists. App assets still live in `assets/`.
